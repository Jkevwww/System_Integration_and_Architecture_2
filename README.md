# System Integration and Architecture 2

Name: Jerome Kevin D. Fiel


Course/Section: BSIT 3-A


Subject: System Integration and Architecture 2

A collection of Laravel activities demonstrating core web development concepts including routing, controllers, Blade templating, form handling, validation, CRUD operations, database integration, authentication, and API integration. These projects showcase the use of Laravel’s MVC architecture to build dynamic and functional web applications with unique systems, database management, user authentication, and external API consumption.


List and explanation of the activities included
Please clearly emphasize and label the following:

SIA_Activity_2 - This activity requires you to create a simple Laravel mini-system with your own unique theme. The system must use routes, controllers, Blade templates, and data passing. You need to create a list page that displays all items and a detail page that shows complete information for a selected item. The project should include at least 5 custom records with 3 attributes each, along with proper navigation between pages using Laravel’s MVC structure.

SIA_Activity_3 - This activity requires students to build a complete Laravel CRUD (Create, Read, Update, Delete) system connected to a MySQL database. Students must configure the database in the .env file, create migrations for the database table, and develop a unique system with full CRUD functionality using controllers, routes, and Blade templates. The project should allow users to add, view, edit, and delete records dynamically from the database while following Laravel’s MVC structure.

SIA_Activity_4 - This activity requires students to create a unique Laravel form system that collects user input through an HTML form using Blade templates. The system must handle form submission with routes and a controller, apply validation rules to ensure correct input, and display validation errors properly. Students must include at least five input fields, use CSRF protection, and make the form retain old input values when validation fails. The goal is to demonstrate Laravel form handling, validation, and error display using the MVC structure.

SIA_Activity_5 (integration-app) - This activity requires students to build a Laravel application that integrates authentication and APIs. Students will set up login and registration using Laravel Breeze, enhance registration with custom fields such as Full Name and Role, and implement role-based dashboards. They will also create their own REST API to return user data and consume data from a public API like Weather API or JSONPlaceholder. The final system should combine authenticated user information, data from the custom API, and external API data into a single dashboard, demonstrating Laravel authentication, API creation, API consumption, and system integration concepts.
